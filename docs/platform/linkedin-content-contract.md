# LinkedIn Content Contract

This document defines the layout, style, formatting, and structural constraints for content generated specifically for LinkedIn.

---

## 1. Structural Constraints

All LinkedIn post generations must strictly conform to these limits:

| Parameter | Constraint | Rationale |
| :--- | :--- | :--- |
| **Max Character Count** | 1,800 characters (soft limit), 3,000 characters (hard API limit) | Shorter, highly-focused posts perform better; reader fatigue sets in quickly. |
| **Paragraph Length** | Max 3 lines per paragraph | Optimizes readability on mobile devices (prevents "walls of text"). |
| **Total Hook Lines** | Max 2 lines before the "...see more" fold | Must capture attention immediately in the feed. |
| **Hashtags** | 3 to 5 relevant hashtags at the very bottom | Helps categorization without cluttering the main content. |
| **Call to Action (CTA)** | Exactly 1 closing question or prompt | Encourages comments, which boosts the LinkedIn algorithm. |

---

## 2. Tone and Style Rules

* **Technical Authority**: Write as a peer sharing insights, not a marketer selling a tool. Use clear, active voice.
* **Technical Breadcrumbs**: Provide exact equations (in text or simplified form), variable names, or architecture components.
* **Formatting Conventions**:
  * Use bold text selectively for emphasis (e.g., **Key takeaway:**).
  * Use standard Unicode bullet points (`*` or `-`) for lists. Do not use custom emojis as bullet points.
  * Leave a blank line between every paragraph and list.

---

## 3. Post Anatomy

A standard LinkedIn post generated by this platform must follow this structure:

```text
[Hook — Line 1: Clear technical problem or insight]
[Hook — Line 2: The novel solution / core paper finding]

[Whitespace]

[Section 1: The Context/Problem — why standard approaches fail]

[Whitespace]

[Section 2: The Core Mechanism — how the new approach works (with figures/values)]

[Whitespace]

[Section 3: The Results/Trade-offs — concrete evaluation metrics]

[Whitespace]

[CTA — A specific technical question for discussion]

[Whitespace]

[Source Citation Block — Paper title, authors, year]
[Link block — e.g. "Primary source link in comments"]

[Whitespace]

[Hashtags]
```

## 4. Examples and Anti-Patterns

### 4.1 Good Example

```text
How do you scale transformer context windows to 1 million tokens without running out of GPU memory?

FlashAttention-3 achieves this by exploiting asymmetric memory hierarchies and overlapping GPU tensor core computations with memory transfers.

Standard attention mechanisms scale quadratically with context length, leading to memory bottlenecks. FlashAttention solved this by tiling, but as GPUs evolved, memory transfer speeds couldn't keep up with Tensor Core throughput.

Here is how FlashAttention-3 addresses this bottleneck:
* Producer-Consumer Warps: Dedicated warps load data into Shared Memory (SMem) while consumer warps perform matrix multiplication directly from SMem.
* Asymmetric Overlap: Computations are scheduled dynamically to hide latency, ensuring tensor cores are never idle waiting for memory blocks.
* FP8 Quantization: Support for 8-bit floating-point numbers cuts memory bandwidth in half while preserving model accuracy.

The results are significant: FlashAttention-3 achieves up to 75% utilization of the theoretical H100 GPU limits, running 2.4x faster than standard FlashAttention-2.

If you are training large models, are you using FP8 training in your pipelines yet, or are you still bottlenecked by FP16/BF16 memory limits?

Primary source link: https://arxiv.org/abs/2407.00215
Paper: FlashAttention-3: Fast Attention with Asymmetric GPU Multiplexing
Authors: Tri Dao et al., 2024.

#MachineLearning #DeepLearning #CUDA #LLMs #Transformer
```

### 4.2 Bad Example (Anti-Pattern)

```text
🚨🚨 BIGGEST AI BREAKTHROUGH OF THE YEAR!!! 🚨🚨

FlashAttention-3 is finally here and it is going to absolutely blow your mind! 🤯🚀

Say goodbye to slow models and hello to 1 MILLION tokens! This is a complete game-changer for AI developers everywhere!

Here is why it is awesome:
🔥 It is super fast.
🔥 It runs on H100s.
🔥 It uses FP8.

Everyone needs to download this immediately! What do you think about this crazy AI revolution? Let me know below! 👇👇👇

#AI #Tech #Innovation #Future
```
* **Why it fails**: Uses sensationalized language, emoji spam, lacks technical details (no explanation of *how* asymmetric overlapping works), and uses generic hashtags.
