# YouTube Shorts Content Contract

This document defines the layout, pacing, formatting, and structural constraints for generating scripts intended for YouTube Shorts (vertical video format).

---

## 1. Structural Constraints

All generated Shorts scripts must strictly conform to these limits:

| Parameter | Constraint | Rationale |
| :--- | :--- | :--- |
| **Duration** | 50 to 58 seconds (strict target) | Avoids hitting the 60-second cutoff while maximizing retention. |
| **Word Count** | 130 to 150 words total | Matches a conversational speaking pace of 150–160 words per minute. |
| **Pacing** | Fast (average 2.5 words per second) | High information density is critical to prevent viewers from swiping away. |
| **Format** | Three-column table: Visuals \| Audio / SFX \| Spoken Script | Clearly separates production cues from spoken voiceover. |
| **CTA** | Interactive prompt at the very end | Asks viewers to subscribe or check the pinned comment for the full source. |

---

## 2. Audio & Visual Styling Rules

* **Hook (0-3 seconds)**: The script must start with an immediate, high-impact technical question or visual setup. Avoid generic greetings ("Hey guys").
* **Visual Transitions**: Transition cues must be specified every 3–5 seconds (e.g., zoom-in, code snippet highlight, graph overlay).
* **Audio Cues (SFX/BGM)**: Specify background music shifts (e.g., upbeat synth, low ambient drone) and sound effects (e.g., keyboard typing, paper whoosh) to support pacing.
* **Spoken Language**: Write for the ear. Use short sentences, clear transitions, and avoid tongue-twisters or overly complex nested clauses.

---

## 3. Script Layout Format

Every YouTube Shorts script must be structured as a Markdown table with the following columns:

1. **Visuals**: Directions for slides, b-roll, code snippets, or animations.
2. **Audio/SFX**: Background music state and timing of sound effects.
3. **Spoken Script**: The exact words to be spoken by the voiceover artist or generated by Text-to-Speech (TTS).

---

## 4. Examples and Anti-Patterns

### 4.1 Good Example

```markdown
### YouTube Shorts Script: How FlashAttention-3 Overlaps Memory

| Visuals | Audio / SFX | Spoken Script |
| :--- | :--- | :--- |
| **[0:00-0:03]** Zoom in on a complex transformer attention equation. A red line strikes through it. | **[SFX: Loud record scratch]** Upbeat, low-volume synth track begins. | How do you scale transformers to a million tokens without running out of GPU memory? |
| **[0:03-0:08]** Clean diagram showing data loading from HBM to Shared Memory. | **[SFX: Swoosh]** Synth track beats drop. | Usually, GPUs waste cycles waiting for memory to load. FlashAttention-3 fixes this by overlapping memory transfer with computation. |
| **[0:08-0:15]** Split screen showing Producer Warps on the left, Consumer Warps on the right. | **[SFX: Keyboard click]** Music intensifies. | It divides GPU warps. Producer warps load data into Shared Memory while consumer warps compute matrix multiplication simultaneously. |
| **[0:15-0:22]** 3D bar chart showing 75% H100 utilization versus 35% for baseline. | **[SFX: Ding]** Music levels out. | This asymmetric overlap yields up to seventy-five percent hardware utilization on H100 GPUs. |
| **[0:22-0:28]** Text overlay: "Read the paper". Pinned comment arrow animation. | Music fades out slowly. | Want to build faster LLMs? Subscribe, and read the full paper linked in the pinned comment! |
```

### 4.2 Bad Example (Anti-Pattern)

```markdown
Hey everyone! Today I want to show you this super cool new paper called FlashAttention-3.

So basically, attention is really slow. If you want to run big models, it takes forever and uses all your memory.

FlashAttention-3 makes it faster. It does this by using FP8 and some warp things. It is very cool.

Let me know what you think in the comments! Don't forget to like and subscribe!
```
* **Why it fails**:
  * Does not use the three-column script format.
  * Lacks specific visual and audio cues.
  * Word count is too low, pacing is unstructured.
  * Contains conversational filler ("Hey everyone!", "So basically") that wastes valuable hook seconds.
  * Does not detail the technical mechanism.
