# Skill: Phase Closeout

## Name
phase-closeout

## Description
Verifies that a phase is complete and safe to close.

## Goal
Check deliverables, documentation, tests, and open risks before marking a phase finished.

## Procedure
1. Read the phase task list and expected deliverables.
2. Verify every deliverable exists.
3. Check that required validations passed.
4. Note remaining risks and blockers.
5. Summarize readiness for the next phase.

## Constraints
- **Read-only**.
- **No task scope expansion**.
- **Do not mark complete without evidence**.

## Output Format
- **Phase Status**
- **Verified Deliverables**
- **Open Risks**
- **Next Phase Readiness**
