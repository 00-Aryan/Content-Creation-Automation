# Skill: Run Phase

## Name
run-phase

## Description
Coordinates execution of one named phase from the project roadmap.

## Goal
Complete the phase within its declared scope and update the related project-control documents.

## Procedure
1. Read the phase plan and related task cards.
2. Confirm dependencies and scope.
3. Execute only phase-approved work.
4. Run the required validations.
5. Report completion or blockers.

## Constraints
- **Do not cross phase scope**.
- **Do not change frozen scopes without approval**.
- **Do not skip required validation**.

## Output Format
- **Phase**
- **Files Changed**
- **Validations**
- **Status**
